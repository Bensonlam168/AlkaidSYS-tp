# 术语表（Glossary）

> 本文档用于集中定义 AlkaidSYS 设计文档中使用的核心术语，作为后续文档编写与阅读的统一参考。

## 1. 核心业务术语

- **tenant / 租户**：
  - 平台服务的业务实体单位，通常对应一个企业或组织；
  - 在数据库中通过 `tenant_id` 字段标识，是数据隔离的主维度。

- **site / 站点**：
  - 租户下的子实例或业务域，如不同业务线、区域站点等；
  - 在数据库中通过 `site_id` 字段标识，通常与 `tenant_id` 组成联合索引。

- **app / 应用**：
  - 在 AlkaidSYS 上运行的业务应用实例，可以是内置应用或通过插件机制安装的第三方应用；
  - 与“插件（plugin）”概念区分：应用是业务交付单元，插件是扩展能力单元。

- **plugin / 插件**：
  - 按统一规范开发、可安装到平台中的扩展模块；
  - 可为一个或多个应用提供功能扩展，如工作流引擎、报表引擎等。

## 2. 低代码与数据建模术语

- **collection / 低代码业务集合**：
  - 低代码平台中抽象出来的业务数据集合，相当于一张业务表或一组相关表；
  - 由字段（field）、关系（relationship）等元数据定义，最终映射到物理数据库表。

- **field / 字段**：
  - Collection 中的单个数据项定义，对应数据库表中的一列；
  - 可包含数据类型、验证规则、UI 显示配置等元信息。

- **relationship / 关系**：
  - Collection 与 Collection 之间的关联关系，如一对多、多对多等；
  - 在数据库中通常通过外键或中间表实现。

## 3. 工作流相关术语

- **workflow / 工作流**：
  - 将一系列业务步骤（节点）按一定条件和顺序组织起来的流程定义；
  - 通常包含触发器（trigger）、节点（node）、连线（edge）、全局变量等要素。

- **approval workflow / 审批流**：
  - 以人工审批节点为主的工作流场景，如费用报销、合同审批等；
  - 节点主要由“提交人、审批人、抄送人”等角色构成。

- **automation workflow / 自动化工作流**：
  - 以系统自动任务为主的工作流场景，如定时任务、数据同步、Webhook 调用等；
  - 节点主要由“API 调用、脚本执行、条件判断”等组成。

- **execution / 工作流执行实例**：
  - 某个工作流定义在特定数据和上下文下的一次实际运行；
  - 通常记录在如 `lowcode_workflow_executions` 等表中，包含状态、起止时间、节点执行结果等。

## 4. 安全与多租户相关术语

- **JWT（JSON Web Token）**：
  - 用于前后端之间传递认证信息的令牌格式，包含签名以防篡改；
  - 在 AlkaidSYS 中用于用户认证和租户/站点上下文传递。

- **RBAC / 基于角色的访问控制**：
  - 通过角色（Role）与权限（Permission）的映射来控制访问；

- **ABAC / 基于属性的访问控制**：
  - 基于用户属性、资源属性和环境属性的策略来决定是否允许访问；

后续如在设计中引入新的关键术语（如分布式事务、读写分离等），建议优先在本文件中补充定义后再在各子文档中引用。
